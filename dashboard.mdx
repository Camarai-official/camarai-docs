## ¿Qué es esto?

El Dashboard es la primera pantalla que ve cualquier admin al entrar. Está diseñado para que, **de un solo vistazo**, puedas saber si el día va bien o mal sin tener que entrar en ninguna sección. ver las métricas clave .

La pantalla se divide en **widgets configurables** — cada uno muestra un aspecto diferente del negocio. Si un widget no te interesa, puedes ocultarlo.

---

## ¿De qué tablas de la base de datos se alimenta?

Para renderizar correctamente todos los widgets, el Dashboard consulta varias tablas de Convex:

| Tabla de Convex | ¿Para qué se usa? |
|---|---|
| `orders` | Calcular ingresos totales, ticket medio, y alimentar el widget de Comandas Recientes |
| `order_items` | Calcular el ranking de productos más vendidos (Top Productos) |
| `ingredients` | Detectar los ingredientes por debajo del stock mínimo (Alertas de Stock) |
| `tables` + `environments` | Calcular la ocupación de mesas y distribución de aforo |
| `staff` | Generar el Ranking de Equipo con ventas por empleado |
| `establishments` | Nombre del restaurante, horario y datos de configuración del encabezado |

> **Nota para el backend:** Todas estas consultas deberían estar en un endpoint de tipo `getDashboardSummary` que reciba `{ from: Date, to: Date }` como parámetro, ya que el DatePicker del dashboard filtra todo por rango de fechas.

## 🪟 Modal: Configurar Dashboard

Para personalizar tu panel, pulsa el botón ⚙️ en la esquina superior derecha. Se abre el modal de configuración donde activas o desactivas cada widget con un simple toggle.

### Sección 1 — Análisis de Ventas
![Modal Configurar Dashboard (parte superior)](/images/modal-dashboard-config-top.png)

| Widget | ¿Qué muestra? | Tabla |
|---|---|---|
| **Métricas Principales** | Ingresos totales, ticket medio, variación vs. semana anterior | `orders` |
| **Ventas por Hora** | Curva de demanda hora a hora (¿cuándo viene más gente?) | `orders` |
| **Gráfico de Ingresos** | Comparativa de facturación día a día del período | `orders` |

### Sección 2 y 3 — Operaciones y Rendimiento
![Modal Configurar Dashboard (parte inferior)](/images/modal-dashboard-config-bottom.png)

| Widget | ¿Qué muestra? | Tabla |
|---|---|---|
| **Comandas Recientes** | Los últimos 5-10 pedidos activos en tiempo real | `orders` |
| **Alertas de Stock** | Lista de ingredientes con stock ≤ al mínimo configurado | `ingredients` |
| **Distribución de Aforo** | % de ocupación por zona (Terraza, Salón, Barra) | `tables`, `environments` |
| **Ranking de Equipo** | Leaderboard del personal por ventas gestionadas | `orders`, `staff` |
| **Top Productos** | Los platos más vendidos del período | `order_items`, `products` |
| **Desglose de Costes** | Distribución de gastos fijos vs. variables | `expenses` *(tabla pendiente)* |

## 🚀 Plan de Mejoras (Roadmap Técnico)

### 🎨 Frontend (UI/UX & Sincronización)
- **Persistencia Local**: Los toggles de widgets se reinician al recargar. Implementar `localStorage` como fallback antes de la persistencia en BBDD.
- **Widgets Faltantes**: El widget **"Tasa Conversión Upsell"** está hardcodeado; falta vincularlo al modal de configuración.
- **Atajos de Fecha**: Añadir botones rápidos "Hoy", "Semana", "Mes" para evitar el uso manual del DatePicker en consultas frecuentes.
- **UX de Configuración**: El campo `badgeTable` no tiene representación visual ni toggle en el modal actual.

### ⚙️ Backend (BBDD & Lógica Convex)
- **Tabla Preferences**: Crear una colección `user_dashboard_preferences` para guardar el estado de los widgets por cada usuario/admin.
- **Agregaciones en Tiempo Real**: Optimizar `getDashboardSummary` para que use índices compuestos por `establishment_id` y `created_at` para evitar latencia en rangos largos.
- **Widget de Costes**: Implementar la tabla `expenses` y su lógica de agregación para alimentar el gráfico de rentabilidad.
