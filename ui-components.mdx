## 🧩 Componentes Base Existentes
El proyecto ya cuenta con un sistema de diseño robusto basado en **Radix UI** y **Tailwind CSS**, localizado en `src/components/ui/`.

**Destacados:**
- Interactivos: `button`, `accordion`, `tabs`, `dropdown-menu`, `dialog`, `sheet`.
- Formularios: `input`, `textarea`, `select`, `checkbox`, `radio-group`, `switch`, `date-range-picker`, `color-picker`.
- Visualización: `table`, `chart`, `carousel`, `progress`, `badge`, `avatar`.
- Layout: `card`, `scroll-area`, `separator`, `sidebar`, `menubar`.
- Específicos del Dominio: `floor-plan-canvas`, `action-tile`, tab-views (ej. `billing-tab`, `staff-tab`, `inventory-tab`, `providers-tab`).

## 🚨 Faltantes Identificados (TO-DO)

Aunque el sistema base está completo, para una operativa real de restaurante (POS) **faltan los siguientes modales y flujos UI**:

### Flujos de Caja y Cobro
- **Cobro Dividido (Split Bill)**: Modal para dividir la cuenta equitativamente o por productos entre varios comensales.
- **Asignación de Cliente a Factura**: Modal para buscar o crear un cliente (CRM) e incluir los datos fiscales en el ticket.
- **Arqueo y Cierre de Caja**: Modal específico para ingresar efectivo de apertura/cierre y confirmar descuadres.
- **Devoluciones (Refunds)**: Flujo de autorización (por PIN) para anular o devolver un pedido cobrado.

### Operativa de Sala
- **Unir Tablas (Merge Tables)**: Modal para agrupar mesas temporalmente.
- **Traspaso de Comanda**: Dialog para mover los artículos o la comanda entera a otra mesa.
- **Modificador Libre / Notas de Cocina Avanzadas**: A veces incluido en la selección, pero un modal rápido para teclear texto libre a un plato.

### Promociones y CRM
- **Aplicar Descuento Manual**: Modal que requiera autorización para aplicar % o importe fijo de descuento al total o al artículo.

### KDS (Cocina)
- **Vista Kanban Interactiva**: Para mover comandas de estado arrastrando tarjetas (Drag & Drop) requerirá un ajuste en la UI existente del KDS.
