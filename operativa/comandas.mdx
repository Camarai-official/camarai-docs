## ¿Qué es esta vista?

Las Comandas son **el registro de cada pedido** que se ha creado en el sistema, ya sea por el TPV, el KDS o manualmente. Esta pantalla es el historial de auditoría: puedes buscar cualquier comanda, ver su desglose, editarla si tiene algún error, cambiarle el estado o exportarla.

Piénsalo como el "libro mayor" de la cocina: todas las ventas del restaurante pasan por aquí.

## La tabla de pedidos: cómo navegar

La tabla muestra paginas de **10 pedidos** por defecto, con estas columnas configurables:

| Columna | Editable | Columna DB |
|---|---|---|
| **Nº Orden** | No | `order_number` |
| **Hora** | No | `created_at` (formateado) |
| **Mesa** | Sí (modal editar) | `table_id` |
| **Nombre** | Sí (modal editar) | `client_name` |
| **Total** | No directo, depende de ítems | calculado de `order_items` |
| **Estado** | Sí (menú acción rápida) | `status` |

## 🪟 Modal: Ver Detalles (Ticket)

![Modal Ver Detalles - Ticket](/images/modal-comandas-detalle.png)

Este modal muestra el **ticket completo** tal como se imprimiría en papel. Incluye:

- **Cabecera**: Logo, nombre del restaurante, dirección y teléfono
- **Número de ticket y fecha/hora**
- **Mesa y nombre del cliente**
- **Línea por ítem**: cantidad, nombre del producto y precio total de esa línea
- **Subtotal**, **IVA (21%)** y **TOTAL en negrita**
- **Footer**: mensaje de despedida y URL del restaurante

Dos botones en la parte inferior: **Imprimir** y **Editar** (que abre directamente el modal de edición).

> **Tabla DB necesaria:** Para generar el ticket encabezado se necesita una tabla `establishments` con los campos `name`, `address`, `phone`, `logo_url`.

## 🪟 Modal: Editar Comanda

![Modal Editar Comanda](/images/modal-comandas-editar.png)

Modal de dos columnas para modificar los detalles de la comanda con rapidez:

**Columna izquierda — Ítems del pedido:**
- Buscador de productos (`Buscar producto o plato...`)
- Lista de ítems con controles `−` / `+` y botón de eliminar 🗑️
- Precio unitario visible en cada ítem

**Columna derecha — Detalles del servicio:**
- **Ubicación**: Selector desplegable de mesa (alimentado de la tabla `tables`)
- **Cliente**: Campo de texto con nombre del cliente
- **Notas de Cocina**: Área de texto libre para instrucciones especiales
- **Resumen económico**: Subtotal + campo de **Descuento (%)** + IVA 21% + **Total** actualizado en tiempo real

Dos botones: **Cancelar** y **Guardar Cambios** (que actualiza el documento `orders` y sus `order_items`).

## 🚀 Plan de Mejoras (Roadmap Técnico)

### 🎨 Frontend (UI/UX & Sincronización)
- **Generación de Ticket PDF**: Implementar la lógica cliente (ej: `jspdf` o similar) para que el botón de exportación genere el documento visual.
- **UX de Búsqueda**: El filtro de fecha debe incluir shortcuts de tiempo (Hoy/Ayer/Semana) para agilizar la auditoría operativa.
- **Evolución del CTA**: Añadir un botón flotante "+" o "Nueva Comanda" para permitir la creación manual desde esta vista.
- **Visualización de Staff**: Mostrar el campo `assigned_to` en la tabla para saber quién es responsable de cada pedido de un vistazo.

### ⚙️ Backend (BBDD & Lógica Convex)
- **Auditoría de Cambios**: Implementar un log de historial en `orders_audit` para registrar quién modificó una comanda ya cerrada y por qué.
- **Integración de Pagos**: Crear el endpoint que vincule la comanda con la pasarela de pagos (Stripe/Redsys) para actualizar el `status` automáticamente.
- **Indices de Búsqueda**: Optimizar la búsqueda por `client_name` mediante búsqueda por texto parcial en Convex para mejorar la velocidad en históricos grandes.
