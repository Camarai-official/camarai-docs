## 1. Ecosistema Camarai — 4 apps + 4 agentes

El ecosistema Camarai se estructura en tres capas interconectadas, bajo una premisa fundamental:

> [!IMPORTANT]
> **Regla de Oro Nº1**: El **Dashboard** es el cerebro de gestión centralizada. Configura impresoras, nodos, empleados y reglas de negocio. El **POS y la Carta** son terminales ligeras que consumen esa configuración para ejecutar la operación.

## 3. Vistas — POS (Terminal de Venta)

**Repo**: `pos-template-main` — SPA (Single Page Application) basada en **Vanilla JS ES6**.
Utiliza un sistema de **DOM Swapping** gestionado en `navigation.js` y fragmentos HTML dinámicos (`templates/`).

| # | Vista | Key en router | Estado | Lógica de Renderizado |
|:--|:------|:--------------|:-------|:----------------------|
| 1 | **Caja** (Registro) | `caja` | ✅ Funcional | Inyectada en `index.html` + `render/pos.js` |
| 2 | **Mesas** | `mesas` | ⚠️ Placeholder | `viewPlaceholders` (UI estática) |
| 3 | **KDS** (Cocina/Barra) | `kds` | ⚠️ Placeholder | `viewPlaceholders` (Oculta PDA Sidebar) |
| 4 | **Clientes** | `clientes` | ⚠️ Placeholder | `viewPlaceholders` (Inyectado vía JS) |
| 5 | **Recibos** | `recibos` | ⚠️ Placeholder | `viewPlaceholders` |
| 6 | **Configuración** | `config` | ⚠️ Placeholder | `viewPlaceholders` (Oculta PDA Sidebar) |

**Total POS**: 6 vistas base vinculadas al Router.

**Arquitectura Vanilla**:
- **Router**: El `initRouter` en `navigation.js` captura los clicks del footer y alterna la visibilidad de la sección de productos vs placeholders.
- **Componentes**: No utiliza Custom Elements nativos (`web-components`), sino **módulos funcionales** que inyectan plantillas HTML (`fillTemplate`) para mantener el rendimiento al máximo en terminales táctiles.
- **PDA Mode**: El sistema detecta automáticamente la vista activa para ocultar o mostrar el panel lateral de pedidos (ej. en KDS se oculta para maximizar visibilidad de tickets).

## 4. Vistas — Carta (Menú Digital)

**Repo**: `carta.camarai.es-main` — Next.js / React / TypeScript.
**Estructura**: App de una sola página (`AllComponents.tsx`) que gestiona el estado dinámico del pedido.

| # | Sección/Componente | Función | Lógica Técnica |
|:--|:-------------------|:--------|:---------------|
| 1 | `Header` | Identidad + Badge de pedido | Muestra nº items en `pedido` state |
| 2 | `CategoryTabs` | Navegación por categorías | Filtra `menuData` en el cliente |
| 3 | `MenuContainer` | Grid de productos y variantes | Renderiza `ProductCard` con +/- y selectores de variante |
| 4 | `CheckoutBar` | Resumen flotante (Sticky) | Acción de cierre de pedido |
| 5 | `CheckoutScreen` | Pantalla de confirmación final | Recoge datos y activa `generarComanda` |
| 6 | `Notification` | Alertas de sistema (Toast) | Feedback de éxito/error en pedido |
| 7 | `Loader` | Pantalla de carga (Splash) | Aparece mientras `obtenerCarta` está en curso |

**Flujo Técnico Verificado**:
1. **Acceso**: 🤖 Agente Cliente (WhatsApp) envía link `carta.camarai.es?session=ID`.
2. **Carga**: La app llama a `/api/obtenerCarta` (Next API Route) -> Proxy a Webhook n8n en `bot1.camarai.es:5678/webhook/get-catalog`.
3. **Selección**: El cliente añade productos al estado local `pedido`.
4. **Envío**: La app llama a `/api/generarComanda` -> Proxy a Webhook n8n en `bot1.camarai.es:5678/webhook/generate-order`.
5. **Cierre**: El pedido entra en el ecosistema Convex y se notifica al POS/Dashboard.

## 7. Decisiones de Arquitectura

<Tabs>
  <Tab title="Filosofía y Stack">
    ```
    Dashboard = Next.js / React / TypeScript  →  componentes .tsx
    POS + PDA = Vanilla JS / HTML / CSS       →  templates .html + .js
    Carta     = Next.js / React / TypeScript  →  componentes .tsx
    Agentes   = n8n + Evolution API + AI      →  flujos backend (no frontend)

    Los componentes React del Dashboard NUNCA van al POS/PDA.
    Los templates HTML del POS NUNCA van al Dashboard/Carta.
    Los agentes NO tienen UI propia — usan WhatsApp, voz o el chat del Dashboard.
    ```

    | Motivo (Vanilla JS) | Explicación |
    |:-------|:------------|
    | **Sin compilación** | El POS se copia como carpeta estática a cualquier dispositivo y funciona. No necesita `npm build`. |
    | **Cualquier dispositivo** | Tablets Android, iPads, Chromebox, terminales táctiles — un navegador basta. |
    | **Rendimiento** | Un POS necesita respuesta instantánea. Vanilla JS no carga virtual DOM ni reconciliación. |
    | **Mantenimiento** | El equipo de cocina/sala no mantiene repos React. Se edita un archivo y listo. |

    ### 7.1 Filosofía de Autoridad (Regla Nº1)
    *   **Dashboard**: Creador de datos (CMS, Configuración técnica, RRHH, Permisos).
    *   **POS / PDA**: Consumidor de datos (Flujo de venta, KDS, Toma de pedidos). No configuran, ejecutan.
    *   **Convex**: El canal que transporta la autoridad del Dashboard a los terminales en tiempo real.
  </Tab>
  <Tab title="Mapa de Roles">
    | | **POS** | **Dashboard** | **Carta** | **PDA** | **Agentes** |
    |:--|:---|:---|:---|:---|:---|
    | **Función** | Vender, cobrar, comandar | Configurar, analizar, gestionar | Menú digital, recibir pedidos | Tomar pedidos en sala | Comunicar, recomendar, automatizar |
    | **Quién lo usa** | Cajeros, cocina | Managers, dueños | Clientes | Camareros | Todos (vía WhatsApp/voz/chat) |
    | **Tipo operación** | 🔥 Escritura intensiva | 📖 Lectura + CRUD | 📖 Lectura + 1 escritura | 🔥 Escritura pedidos | 🤖 Lee + escribe según agente |
    | **Dónde corre** | Tablet/terminal | PC/portátil | Móvil cliente | Móvil camarero | Servidor (n8n) |
  </Tab>
  <Tab title="Conexión a Datos">
    **La única capa compartida es el backend Convex.**

    | Capa | ¿Compartida? | Formato |
    |:-----|:-------------|:--------|
    | Componentes UI | ❌ No | `.tsx` (React) vs `.html/.js` (Vanilla) |
    | Lógica de negocio | ❌ No | Hooks vs `core/*.js` |
    | **Types / Interfaces**| ✅ **SÍ** | TypeScript `.ts` (Paquete compartido) |
    | **Schema Convex** | ✅ **SÍ** | `convex/schema.ts` (Único) |
    | **Queries/Mutations** | ✅ **SÍ** | `convex/functions/*.ts` (Único) |

    ---

    | Producto | SDK / Conexión | Tipado |
    |:---------|:---------------|:-------|
    | **Dashboard** | `convex/react` | TypeScript nativo |
    | **POS / PDA** | `convex/browser` | JS puro (Documentation-based) |
    | **Carta** | `convex/react` / `fetch` | TypeScript nativo |
    | **Agentes** | Convex HTTP API | Nulo (n8n JSON nodes) |
  </Tab>
