# Ecosistema de Autenticación

Camarai utiliza una estrategia de autenticación híbrida diseñada para la máxima seguridad en el Dashboard y la máxima agilidad en la sala del restaurante. El pilar central de esta estrategia es **Camarauth**.

---

## Matriz de Selección de Método
¿Cuál usar y cuándo? Esta tabla define el canal oficial según el perfil y dispositivo.

| Canal | Método | Dispositivo | Caso de Uso |
| :--- | :--- | :--- | :--- |
| **Administración** | Email / Password | PC / Laptop | Configuración profunda y finanzas. |
| **Sala (Rápido)** | PIN (4 dígitos) | Tablet / PDA | Cambio de camarero en mitad de servicio. |
| **Agentes / Móvil** | **Camarauth** (WA) | Smartphone | Notificaciones críticas y acceso remoto. |
| **Clientes** | WhatsApp Link | Smartphone | Acceso a Carta Digital y Reservas. |

## Qué es Camarauth
Camarauth no es un simple redireccionamiento; es un ecosistema de librerías para Node.js y React que gestionan el ciclo de vida de la sesión en tiempo real.

## Integración Técnica del SDK

### 1. El Hook Maestro: `usePinAuth`
Para el desarrollo de interfaces (Dashboard y POS), el SDK proporciona los hooks necesarios para gestionar el estado de la autenticación sin boilerplate adicional.

```tsx
const { 
  pin,           // El código numérico generado
  emojis,        // Representación visual amigable (🎭)
  whatsappLink,  // URL para abrir WhatsApp con el mensaje pre-llenado
  qrCodeUrl,     // QR para escaneo rápido desde escritorio
  generate       // Función para disparar la creación de un nuevo PIN
} = usePinAuth({ 
  apiUrl: 'http://localhost:3001',
  whatsappNumber: '+34XXXXXXX' 
});
```

### 2. Backend: `CamarauthBackend`
El servidor actúa como puente entre la aplicación y la **Evolution API** (WhatsApp), gestionando WebSockets con **Socket.io** para una respuesta instantánea.

- **Configuración Requerida**:
  - `evolutionApiUrl`: Dirección de la pasarela WhatsApp.
  - `evolutionApiKey`: Credencial de acceso.
  - `evolutionInstanceName`: Instancia específica vinculada al local.

## Características Únicas del SDK

### PIN Encoding (Emojis)
Una de las innovaciones de Camarauth es la codificación de PINs en emojis. Esto mejora la experiencia de usuario (UX) al hacer que el código sea visualmente memorable y difícil de confundir con otros números en pantalla.

### Auto-Regeneración y Tiempo Real
- **WebSocket**: El cliente (React) se suscribe a un canal de Socket.io. En cuanto el usuario pulsa el link en WhatsApp, el backend recibe la notificación de la Evolution API y la comunica al frontend en **milisegundos**.
- **Sesiones Seguras**: Genera JWTs (JSON Web Tokens) con mecanismos de **auto-refresh** para mantener la sesión activa sin fricciones innecesarias.
